## HighLevel + Notion operating stack

Use Notion as the editorial and research system. Use HighLevel as the default lead-capture, CRM, nurture, and pipeline layer behind the blog.

### Default stack assumptions
- topic strategy, briefs, and calendar planning live in Notion
- blog CTAs route into HighLevel forms, surveys, funnels, website pages, or calendars
- contacts, tags, custom fields, opportunities, and attribution live in HighLevel
- email and SMS nurture run through HighLevel workflows
- distribution queues can use HighLevel Social Planner when the user wants execution inside HighLevel

### Strategy output requirement
Every strategy document should include:
- the recommended HighLevel capture path for each content pillar
- the pipeline stage or conversion goal tied to each CTA
- follow-up workflow notes for email, SMS, or appointment booking
- any Social Planner, blog, funnel, or calendar dependencies needed to execute inside HighLevel

### API usage
If location access exists, use the shared `highlevel-api` skill to inspect or create blogs, authors, categories, posts, forms, surveys, calendars, contacts, opportunities, tags, custom fields, or Social Planner assets.
